Angular

Angular est une réécriture complète d’Angular JS, cardriciel et construit par la même équipe.

Angular créé en 2009 par Misko Hevery est un framework frontal open source, très populaire conçu pour les applications Web modernes et puissantes. Il continue d'affirmer sa domination en tant que nom le plus bien établi dans le monde des frameworks JavaScript front-end. Ce framework est compétent pour développer des applications monopages puissantes et vieillissantes. Voulant utiliser angular pour créer une application, on fait appel aux typescrits, le typescript est un sous ensemble de javascript. Angular formes de différent composante ou l’anglais component, il fonctionne sur plusieurs plateformes mobile et desktop, est connu pour sa rapidité, sa productivité et surtout la valeur qu’apporte typescript. Angular élimine le code supplémentaire que les développeurs devraient généralement écrire en tirant parti de l'injection de dépendances et de la liaison de données, il permet aux développeurs d'utiliser HTML.

Angular est meilleur pour le développement Web ?

raison de sa popularité est qu'il est facile et simple à utiliser, il simplifie également le processus de développement et la structure du code JavaScript. Pour réaliser une application mobile avec Angular, il est nécessaire de l’utiliser avec un autre framework tel que NativeScript ou Ionic, ces frameworks permettent la création d’applications mobiles multiplateformes, le framework Angular est considéré comme une technologie pérenne, réputé difficile, l’apprentissage de cette plateforme de développement pourrait rebuter les développeurs en herbe. Son utilisation s’adresse en effet plutôt à des développeurs qui utilisent déjà TypeScript ou JavaScript, et qui ont de solides connaissances en HTML et en CSS.

Le composant ou component Angular

de base d’une application Angular, les composants, aussi appelés components, représentent toute la partie visuelle de l’application. Un composant peut être une page entière, un élément ou un sous-élément d’un menu, un en-tête, un pied de page, ou bien encore une boîte de dialogue.

component est constitué de plusieurs éléments :
•
fichier de template, sous forme de fichier HTML, qui est en quelque sorte le moule de l’interface utilisateur;
•
un fichier de style CSS, permettant de personnaliser l’habillage visuel du composant; •
fichier TypeScript qui définit le comportement du composant et gère toute la logique du rendu visuel de l’application.
